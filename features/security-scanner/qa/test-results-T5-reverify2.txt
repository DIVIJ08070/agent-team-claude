  - report.md remediation body ... the DETERMINISTIC TEMPLATE, not AI-authored

       tags; call B runClaudeRemediation returned null → template fallback)

ROOT CAUSE (pinned + reproduced — this is a GAP, not restraint, and NOT caused
by BUG-007's now-fixed snippets):

  The pipeline spawns `claude -p` under config.MINIMAL_ENV, whose allowlist is
  {PATH, HOME, LANG}. The Claude Code CLI's macOS Keychain login lookup ALSO
  needs the `USER` environment variable, which MINIMAL_ENV omits. So every real
  scan's `claude -p` returns:

    {"type":"result","is_error":true,
     "result":"Not logged in · Please run /login",
     "total_cost_usd":0,"usage":{"input_tokens":0,...}}

  parse.textFromEnvelope() maps is_error:true → null → parseEnrichResponse null
  → enrich ok:false, findingCount 0 (enrich.ts:110 branch); the same for call B
  → markdown null → template body. degraded stays false because a zero-finding
  toolRun is recorded (not a skip).

DIRECT BISECTION PROOF (echo hi | claude -p --output-format json):
  full shell env ......................... is_error:false, cost $0.059, 4417 tok  ✅ logged in
  full env MINUS ANTHROPIC_API_KEY ....... is_error:false, 4417 tok              ✅ (NOT the API key)
  env -i PATH+HOME+LANG (= MINIMAL_ENV) .. "Not logged in · Please run /login"    ❌
  env -i PATH+HOME+LANG + USER ........... is_error:false                          ✅ USER is the missing var
  env -i PATH+HOME+LANG + LOGNAME ........ "Not logged in"                          ❌
  Notes: ANTHROPIC_API_KEY is NOT set in the shell (login is Claude Code OAuth,
         not an API token — consistent with the PRD). No ~/.claude/.credentials.json
         → credential lives in the macOS Keychain; the CLI keys its lookup on USER.

IMPACT: the AI half of the "hybrid scanner + local Claude" product — AI
enrichment, logic-flaw newFindings, and the AI-authored remediation guide — is
silently non-functional on EVERY real scan. The scanner half is fully correct
and non-degraded. Filed as BUG-008 (HIGH).
